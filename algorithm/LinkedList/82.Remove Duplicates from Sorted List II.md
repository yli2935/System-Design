# 82. Remove Duplicates from Sorted List II（删除排序链表中的重复元素 II）

> LeetCode 82 | 难度：Medium  
> 时间复杂度：O(n) | 空间复杂度：O(1)

---

## 一、问题描述

给定一个 **已按升序排列** 的链表头结点 `head`，删除所有 **数值出现过重复** 的结点（重复数字对应的结点 **一个都不留**），只保留在原链表中 **只出现一次** 的结点。

**与 83 题的区别**：83 是「重复值只保留一个」；本题是「只要该值出现过多次，带这个值的所有结点全部删掉」。

```
输入：head = [1,2,3,3,4,4,5]
输出：[1,2,5]

输入：head = [1,1,1,2,3]
输出：[2,3]

输入：head = [1,1,2,2]
输出：[]
```

---

## 二、核心思想：哑结点 + 前驱不进入「重复段」

链表已排序，**相同值必然连续**。若 `cur.val == cur.next.val`，则从 `cur` 开始整段同值结点都要跳过。

### 两个指针角色

| 指针 | 含义 |
|------|------|
| `prev` | **上一段合法链**的尾结点；其 `next` 将接到「下一段」的第一个结点 |
| `cur` | 当前考察的起点 |

### 分支逻辑

1. **`cur` 与 `cur.next` 同值** → 进入重复段：记下值 `x`，不断 `cur = cur.next` 直到值不是 `x`；然后 **`prev.next = cur`**，把前驱直接连到重复段之后。**注意**：此时 **不要把 `prev` 移到刚跳过的结点**，因为新接上的 `cur` 可能又是另一段重复的开头。
2. **否则** → `cur` 是「只出现一次」的结点：`prev = cur`，`cur = cur.next`，正常往后走。

### 为什么要 dummy？

头结点可能被整段删掉（例如 `[1,1,2]`），用 `dummy.next` 表示新头，避免对 `head` 特判。

## 三、完整代码

```java
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public ListNode deleteDuplicates(ListNode head) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;
        ListNode prev = dummy;
        ListNode cur = head;

        while (cur != null) {
            if (cur.next != null && cur.val == cur.next.val) {
                int x = cur.val;
                while (cur != null && cur.val == x) {
                    cur = cur.next;
                }
                prev.next = cur;
            } else {
                prev = cur;
                cur = cur.next;
            }
        }
        return dummy.next;
    }
}
```

---

## 四、执行过程示例

`head = 1 → 2 → 3 → 3 → 4 → 4 → 5`

| 阶段 | prev（值） | cur（起点） | 操作 |
|------|------------|-------------|------|
| 初值 | dummy | 1 | 1≠2，`prev`→1，`cur`→2 |
| | 1 | 2 | 2≠3，`prev`→2，`cur`→3 |
| | 2 | 3 | 3=3，跳过所有 3，`prev.next`→4，`cur`→4 |
| | 2 | 4 | 4=4，跳过所有 4，`prev.next`→5，`cur`→5 |
| | 2 | 5 | 无 next，`prev`→5，`cur`→null |

最终：`dummy → 1 → 2 → 5`，返回 `dummy.next`。

---

## 五、边界情况

| 情况 | 行为 |
|------|------|
| `head == null` | 不进入循环，返回 `dummy.next` 即 `null` |
| 全表同值如 `1→1→1` | 一次跳过，`prev.next = null`，返回空链表 |
| 重复只在中间 | `prev` 始终在「已确认唯一」的尾上，跳过中间段即可 |
| 无重复 | 每次走 `else`，等价于扫一遍，`prev` 沿链后移 |

---

## 六、总结

| 要点 | 内容 |
|------|------|
| **技巧** | 哑结点 + `prev` 停在「安全尾」，重复段只改 `prev.next`、不前进 `prev` |
| **识别重复** | 排序链表：`cur.val == cur.next.val` 即进入重复段 |
| **易错点** | 跳过重复后 **不要** 误把 `prev` 设成重复段内结点；下轮继续用当前 `prev` 接新的 `cur` |
