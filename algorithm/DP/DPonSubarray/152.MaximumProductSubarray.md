# 152. Maximum Product Subarray（最大子数组乘积）

> LeetCode 152 | 难度：Medium  
> 时间复杂度：O(n) | 空间复杂度：O(1)

---

## 一、问题描述

给定一个整数数组 `nums`，找出数组中乘积最大的非空连续子数组，返回该子数组的乘积。

```
输入：nums = [2, 3, -2, 4]
输出：6
解释：子数组 [2, 3] 有最大乘积 6

输入：nums = [-2, 0, -1]
输出：0
解释：结果不能为 2，因为 [-2, -1] 不是连续子数组
```

---

## 二、为什么不能直接套用 53 题的思路？

### 乘积 vs 求和的关键差异

| 特性 | 求和 | 乘积 |
|------|------|------|
| 负数影响 | 负数只会减小和 | 负 × 负 = 正，可能变最大 |
| 零的影响 | 零只是不增加 | 零会让乘积归零 |
| 单调性 | 前缀和单调变化 | 乘积可能剧烈波动 |

**核心问题**：当前最小乘积，乘以一个负数，可能变成最大乘积！

---

## 三、核心思想：同时追踪最大和最小

### 关键洞察

在每个位置，我们需要同时维护：
- `maxProduct`：以当前位置结尾的 **最大** 乘积
- `minProduct`：以当前位置结尾的 **最小** 乘积（可能是负数）

为什么要记录最小值？
> 因为一个很小的负数，乘以下一个负数，可能变成很大的正数！

### 状态转移

对于每个 `nums[i]`，有三种选择：
1. 从 `nums[i]` 重新开始
2. 继承之前的最大值：`maxProduct * nums[i]`
3. 继承之前的最小值：`minProduct * nums[i]`

```
newMax = max(nums[i], maxProduct * nums[i], minProduct * nums[i])
newMin = min(nums[i], maxProduct * nums[i], minProduct * nums[i])
```

---

## 四、变量含义

| 变量 | 含义 |
|------|------|
| `maxProduct` | 以当前位置结尾的最大乘积 |
| `minProduct` | 以当前位置结尾的最小乘积 |
| `result` | 全局最大乘积（最终答案） |

---

## 五、完整代码

### 方法一：直接取三者最值

```javascript
function maxProduct(nums) {
    let maxProduct = nums[0];
    let minProduct = nums[0];
    let result = nums[0];
    
    for (let i = 1; i < nums.length; i++) {
        const n = nums[i];
        
        // 计算三个候选值
        const candidates = [n, maxProduct * n, minProduct * n];
        
        // 更新最大和最小
        maxProduct = Math.max(...candidates);
        minProduct = Math.min(...candidates);
        
        // 更新全局最大
        result = Math.max(result, maxProduct);
    }
    
    return result;
}
```

## 六、执行过程示例

以 `nums = [2, 3, -2, 4]` 为例：

| i | n | maxProduct | minProduct | result |
|---|---|------------|------------|--------|
| 0 | 2 | 2 | 2 | 2 |
| 1 | 3 | 6 | 3 | **6** |
| 2 | -2 | -2 | -12 | 6 |
| 3 | 4 | 4 | -48 | 6 |

最终答案：**6**

---

以 `nums = [-2, 3, -4]` 为例（负负得正）：

| i | n | maxProduct | minProduct | result |
|---|---|------------|------------|--------|
| 0 | -2 | -2 | -2 | -2 |
| 1 | 3 | 3 | -6 | 3 |
| 2 | -4 | 24 | -12 | **24** |

最终答案：**24**（整个数组 -2 × 3 × -4 = 24）

## 七、边界情况处理

### 1. 包含零

```
nums = [-2, 0, -1]
```
- 零会"切断"乘积链
- 遇到零后，`maxProduct` 和 `minProduct` 都变成 0
- 下一个元素会重新开始

### 2. 全是负数

```
nums = [-2, -3, -4]
```
- 追踪 `minProduct` 的重要性体现
- `-2 × -3 = 6`，然后 `6 × -4 = -24`（变最小）
- 但 `minProduct = -24`，如果后面还有负数，可能翻盘

### 3. 单个元素

```
nums = [-5]
```
- 直接返回 `-5`

## 八、与 53 题对比

| 方面 | 53. 最大子数组和 | 152. 最大子数组乘积 |
|------|-----------------|-------------------|
| 核心变量 | `current_sum`, `min_prefix` | `maxProduct`, `minProduct` |
| 状态数量 | 1 个（当前和） | 2 个（最大、最小乘积） |
| 负数处理 | 无需特殊处理 | 需要同时追踪最小值 |
| 零的影响 | 累加 0 不影响 | 乘 0 归零，需重新开始 |
| 思维模型 | 前缀和差值 | 动态规划 + 双状态 |

## 九、总结

1. **核心难点**：负数乘负数可能得正数，所以必须同时追踪最大和最小
2. **状态定义**：
   - `maxProduct[i]`：以 i 结尾的最大乘积
   - `minProduct[i]`：以 i 结尾的最小乘积
3. **转移方程**：
   ```
   maxProduct = max(nums[i], maxProduct * nums[i], minProduct * nums[i])
   minProduct = min(nums[i], maxProduct * nums[i], minProduct * nums[i])
   ```
4. **优化技巧**：遇到负数时，先交换 max 和 min，可简化代码
5. **空间优化**：只需要前一个位置的状态，O(1) 空间即可
