# 128. Longest Consecutive Sequence（最长连续序列）

> LeetCode 128 | 难度：Medium  
> 时间复杂度：O(n) | 空间复杂度：O(n)

---

## 一、问题描述

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列的长度。

**要求**：时间复杂度 O(n)

```
输入：nums = [100, 4, 200, 1, 3, 2]
输出：4
解释：最长连续序列是 [1, 2, 3, 4]，长度为 4

输入：nums = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]
输出：9
解释：最长连续序列是 [0, 1, 2, 3, 4, 5, 6, 7, 8]，长度为 9
```

---

## 二、为什么不能排序？

排序的时间复杂度是 O(n log n)，题目要求 O(n)。

**关键问题**：如何在 O(n) 时间内找到连续序列？

答案：**HashSet**

---

## 三、核心思想：HashSet + Remove（边扩展边删除）

### 朴素思路（会超时）

```javascript
// 对于每个数，向上找连续的数
for (let num of nums) {
    let count = 1;
    while (set.has(num + count)) {
        count++;
    }
    maxLen = Math.max(maxLen, count);
}
```

问题：如果数组是 `[1, 2, 3, 4, 5]`，从 1 开始找到 5，从 2 开始又找到 5... **重复计算！**

### 优化：边扩展边删除

**关键洞察**：用 `remove()` 代替 `has()`，处理过的元素直接从集合中删除，保证每个元素只被访问一次。

```
以 num = 3 为例，序列 [1, 2, 3, 4, 5]：

向左扩展：remove(2) ✓ → remove(1) ✓ → remove(0) ✗
向右扩展：remove(4) ✓ → remove(5) ✓ → remove(6) ✗

当前序列长度 = 1 + 2(左) + 2(右) = 5
```

当后续遍历到 1, 2, 4, 5 时，它们已经被删除，直接跳过。

**每个元素只被删除一次**，总时间 O(n)。

---

## 四、算法流程

```
1. 将所有数字放入 HashSet
2. 遍历原数组每个数字：
   - 初始化 current = 1
   - 向左扩展：while(set.remove(num-1)) → current++
   - 向右扩展：while(set.remove(num+1)) → current++
   - 更新 max
3. 返回最大长度
```

**核心**：`remove()` 返回 `true` 表示元素存在并被删除，返回 `false` 表示不存在。

---

## 五、完整代码

### Java 版本

```java
class Solution {
    public int longestConsecutive(int[] nums) {
        HashSet<Integer> set = new HashSet<Integer>();
        int max = 0;
        
        // Step 1: 将所有数字放入 HashSet
        for (int num : nums) {
            set.add(num);
        }
        
        // Step 2: 遍历每个数字，双向扩展
        for (int num : nums) {
            int prev = num - 1;    // 向左扩展的指针
            int next = num + 1;    // 向右扩展的指针
            int current = 1;       // 当前序列长度
            
            // 向左扩展，边删除边计数
            while (set.remove(prev--)) {
                current++;
            }
            
            // 向右扩展，边删除边计数
            while (set.remove(next++)) {
                current++;
            }
            
            max = Math.max(max, current);
        }
        
        return max;
    }
}
```

### JavaScript 版本

```javascript
function longestConsecutive(nums) {
    const set = new Set(nums);
    let max = 0;
    
    for (const num of nums) {
        let prev = num - 1;
        let next = num + 1;
        let current = 1;
        
        // 向左扩展
        while (set.delete(prev)) {
            prev--;
            current++;
        }
        
        // 向右扩展
        while (set.delete(next)) {
            next++;
            current++;
        }
        
        max = Math.max(max, current);
    }
    
    return max;
}
```

> **注意**：Java 的 `set.remove()` 和 JavaScript 的 `set.delete()` 都返回布尔值，表示元素是否存在并被删除。

## 六、执行过程示例

以 `nums = [100, 4, 200, 1, 3, 2]` 为例：

**Step 1**：构建 HashSet
```
set = {100, 4, 200, 1, 3, 2}
```

**Step 2**：遍历原数组，双向扩展

| num | 向左 remove | 向右 remove | set 状态 | current | max |
|-----|------------|------------|----------|---------|-----|
| 100 | 99? ✗ | 101? ✗ | {4,200,1,3,2} | 1 | 1 |
| 4 | 3✓→2✓→1✓→0? ✗ | 5? ✗ | {200} | 4 | **4** |
| 200 | 199? ✗ | 201? ✗ | {} | 1 | 4 |
| 1 | (已删除，跳过) | - | {} | 1 | 4 |
| 3 | (已删除，跳过) | - | {} | 1 | 4 |
| 2 | (已删除，跳过) | - | {} | 1 | 4 |

**详细过程**：

1. **num = 100**：左边 99 不存在，右边 101 不存在 → current = 1
2. **num = 4**：
   - 向左：remove(3)✓ → remove(2)✓ → remove(1)✓ → remove(0)✗
   - 向右：remove(5)✗
   - current = 1 + 3 = 4 ✓
3. **num = 200**：左右都不存在 → current = 1
4. **num = 1, 3, 2**：已被删除，remove 返回 false，current = 1

最终答案：**4**

## 七、时间复杂度分析

**为什么是 O(n) 而不是 O(n²)？**

虽然有嵌套循环，但：

- 外层循环：遍历 n 个数
- 内层 while：**每个元素最多被 remove 一次**

**关键**：`remove()` 操作保证每个元素只被处理一次！

```
set.remove(x) 的特性：
- 元素存在 → 删除并返回 true（只发生一次）
- 元素不存在 → 返回 false
```

总 remove 操作次数 ≤ n，所以时间复杂度是 **O(n)**。

### 与"起点判断"方法对比

| 方法 | 避免重复的方式 | 每个元素访问次数 |
|------|--------------|-----------------|
| 起点判断 | 只从 `num-1` 不存在的数开始 | 最多 2 次 |
| Remove | 处理后直接删除 | 恰好 1 次 |

两种方法都是 O(n)，Remove 方法更直观。

## 八、为什么用 Set 而不是 Map？

- Set 足够：只需要 O(1) 判断存在性 + 删除
- Map 多余：不需要存储额外信息

**HashSet 关键操作**：

| 操作 | Java | JavaScript | 复杂度 | 返回值 |
|------|------|------------|--------|--------|
| 添加 | `add(x)` | `add(x)` | O(1) | - |
| 判断 | `contains(x)` | `has(x)` | O(1) | boolean |
| 删除 | `remove(x)` | `delete(x)` | O(1) | **boolean** |

**`remove()` 的返回值是关键**：返回 `true` 表示元素存在并被删除，`false` 表示不存在。

## 十、变体写法：起点判断法

另一种经典 O(n) 解法：只从序列起点开始计数。

```java
class Solution {
    public int longestConsecutive(int[] nums) {
        HashSet<Integer> set = new HashSet<>();
        for (int num : nums) set.add(num);
        
        int max = 0;
        for (int num : set) {
            // 只从起点开始（num-1 不存在）
            if (!set.contains(num - 1)) {
                int current = 1;
                while (set.contains(num + current)) {
                    current++;
                }
                max = Math.max(max, current);
            }
        }
        return max;
    }
}
```

### 两种方法对比

| 方面 | Remove 法 | 起点判断法 |
|------|----------|-----------|
| 避免重复 | 删除已处理元素 | 只从起点开始 |
| 扩展方向 | 双向（左+右） | 单向（只向右） |
| 遍历对象 | 原数组 | Set |
| 复杂度 | O(n) | O(n) |

## 十一、常见错误

### 错误 1：用 has() 而不是 remove()
```java
// ❌ 错误：没有删除，导致重复计算，O(n²)
while (set.contains(prev--)) {
    current++;
}
```

### 错误 2：用排序
```java
// ❌ 错误：O(n log n)，不满足要求
Arrays.sort(nums);
```

### 错误 3：prev/next 更新时机错误
```java
// ❌ 错误：先减再判断，会漏掉 num-1
while (set.remove(--prev)) {  // 应该是 prev--
    current++;
}
```

### 错误 4：只向一个方向扩展
```java
// ❌ 错误：如果从序列中间开始，只向右扩展会漏掉左边的数
while (set.remove(next++)) {
    current++;
}
// 忘记向左扩展！
```

## 十二、总结

| 要点 | 内容 |
|------|------|
| **数据结构** | HashSet |
| **核心技巧** | 双向扩展 + Remove 避免重复 |
| **关键操作** | `set.remove()` 返回是否成功删除 |
| **时间复杂度** | O(n)，每个元素只被删除一次 |
| **空间复杂度** | O(n)，存储所有元素 |

### 核心代码模式

```java
HashSet<Integer> set = new HashSet<>();
for (int num : nums) set.add(num);

for (int num : nums) {
    int prev = num - 1;
    int next = num + 1;
    int current = 1;
    
    while (set.remove(prev--)) current++;  // 向左扩展
    while (set.remove(next++)) current++;  // 向右扩展
    
    max = Math.max(max, current);
}
```

### 核心思想

```
遍历每个数 → 双向扩展 → 边扩展边删除 → 保证每个元素只处理一次
```
